Architecture Overview

Factory Inventory Management System

A full-stack demo for inventory tracking, order management, demand forecasting and spending analytics. Single-page Vue 3 frontend talks to a Python FastAPI backend that serves JSON-file-backed mock data — no database, no auth.

System Architecture

Three layers: browser-rendered SPA, in-process Python API, and on-disk JSON fixtures loaded into memory at startup.

Data Flow — A Filtered Dashboard Request

What happens when the user picks a warehouse from the global filter bar.

  1. User interaction

    User changes a value in FilterBar.vue. The selection mutates a shared ref in composables/useFilters.js — all views see the change reactively.

  2. View reacts

    Each view (e.g. views/Dashboard.vue) watches the filter refs and calls its loader, e.g. api.getDashboardSummary(filters).

  3. API client builds the request

    client/src/api.js uses URLSearchParams and skips any filter whose value is 'all', then issues an axios.get() to http://localhost:8001/api/....

  4. FastAPI routes the call

    Uvicorn hands the request to main.py. CORS middleware allows the cross-origin call. Query params bind to typed function arguments.

  5. Filtering in memory

    Endpoint calls apply_filters() (warehouse, category, status) and filter_by_month() (single months or quarters from QUARTER_MAP) against the in-memory list loaded by mock_data.py.

  6. Pydantic validates

    The result is coerced through response_model (e.g. List[InventoryItem]) before being serialised to JSON. Schema mismatches surface here as 500s.

  7. Render

    The view stores the response in a ref; derived numbers (totals, KPIs) live in computed() properties; the template re-renders. Loading / error states use the standard v-if ladder.

Key Patterns

Conventions worth knowing before changing anything.

Centralised filters

Filter state lives in a module-level ref inside composables/useFilters.js so every view shares the same selection. The literal 'all' is the sentinel for “no filter” on both client and server.

Reactivity contract

  • Raw responses live in ref()
  • Derived values (KPIs, charts) live in computed()
  • v-for keys use stable IDs (sku, order_number) — never the index

In-memory data

mock_data.py reads each JSON file on import and exposes plain Python lists. Mutations don’t persist — restart the server to reset. No DB, no migrations, no async I/O.

Time filtering

The month parameter accepts either a direct ISO month (2025-03) or a quarter key (Q1-2025) resolved via QUARTER_MAP in main.py. Inventory has no time dimension, so it ignores month.

Internationalisation

UI strings flow through useI18n() with locales in client/src/locales/{en,ja}.js. Components call t('nav.companyName') rather than embedding text.

Production gaps

No auth, no rate limiting, CORS *, no input sanitisation beyond Pydantic types, no persistence. Intentional — this is a workshop sandbox.
